Auto Healing microservice to heal Virtual Network Functions:

Overview:

This project was done when I was an upstream contributor to the Virtual Function Controller component of the Open Network Automation Platform project.

The outcome of the project was to provide an Auto Healing endpoint in the Virtual Function Controller to heal Virtual Network Functions (VNFs) running inside VMs/containers.

Languages and Tools:

  • Python Django
  • Docker

Working:
  • The Virtual Function Controller maintain a database for logical mapping between infrastructure resources (Virtual Machines /Containers) with Virtual Functions running inside them.
  • Upon calling the auto healing endpoint to heal a specific Virtual Network function, the Virtual Function Controller looks up the database to find the corresponding infrastructure resource which can be a VM or a container and restart it.
  • This way, the VNF running inside the instance restarts and the virtual function is healed.
  • The endpoint is usually triggered by failure detection such as Consul's service discovery feature wherein the VNFs endpoint is specific in the service discovery and if the endpoint is not responding back, a call to the Auto healing endpoint is done to restart it.
  • Dockerfile for this endpoint was also written.
