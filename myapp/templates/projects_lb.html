Creation of V2.0 REST API in Octavia open source Load Balancer

Octavia Overview Docs: https://docs.openstack.org/octavia/latest/reference/introduction.html

Overview:

This was done when I was an upstream contributor to OpenStack Load Balancer Octavia project.

The effort was mainly aimed at creating V2.0 API endpoints for the load balancer. The V2.0 was created to split the longer endpoint to smaller shorter ones so that clients will need lesser data to manipulate the load balancer REST resources.

The example changed from /v1/loadbalancer/'LB_ID'/pools 'POOL_ID'/member/'MEMBER_ID' to a simpler /v2/loadbalancer, /v2/pool etc. splitting the resources, making client calls simpler.

I was personally responsible for implementing the API endpoint for CRUD operations on a Pool resource where Pool in the load balancer is responsible for logically maintaining the number of member Virtual Machines within which the traffic will be balanced based on the type of algorithm specified.

Languages and Tools:

  • Python
  • Pecan Library
  • Apache HTTP
